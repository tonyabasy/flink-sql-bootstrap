# 安全策略

## 支持版本

我们为最新稳定版本提供安全更新。在报告漏洞之前，请确保你正在运行最新版本。

| 版本     | 是否支持           |
| -------- | ------------------ |
| 最新版   | ✅ |
| 低于最新版 | ❌ |

## 报告漏洞

如果你发现了本项目中的安全漏洞，请负责任地报告。

**首选方式**：使用 [GitHub Security Advisories](https://github.com/tonyabasy/flink-sql-bootstrap/security/advisories/new) 提交私密漏洞报告。这可以确保问题被保密处理并得到妥善跟踪。

**备选方式**：如果你无法使用 GitHub Security Advisories，请新建一个 Issue，并在标题前加上 `[SECURITY]` 前缀。**不要**在公开 Issue 中披露敏感细节——提供高层摘要即可，我们会通过私信与你联系。

## 报告内容

为了帮助我们快速分类和修复问题，请包含以下内容：

- 漏洞的清晰描述及其影响范围
- 复现步骤（最小 SQL 脚本、CLI 参数、环境信息）
- 受影响的版本或 commit 范围
- 任何建议的缓解措施或修复方案（可选）

## 响应时间线

| 阶段     | 时间线              |
| -------- | ------------------- |
| 确认收到 | 收到报告后 7 天内   |
| 初步评估 | 14 天内             |
| 修复与发布 | 视严重程度而定；严重问题优先处理 |
| 公开披露 | 在修复可用后与报告者协调公开 |

## 用户安全最佳实践

- 始终运行 `flink-sql-bootstrap` 的最新发布版本。
- 执行来自不可信来源的 SQL 脚本前请先审查——本工具会编译并提交任意 SQL 到 Flink 集群。
- 在生产环境中使用本工具时，确保你的 Flink 集群已正确安全配置（网络隔离、启用认证）。
- 发现任何可疑行为或意外网络请求时，请及时报告。
